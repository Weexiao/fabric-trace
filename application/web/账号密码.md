需要修改的东西

表单校验缺失：未设置 :rules 与 this.$refs.form.validate，各字段必填/格式（手机号、年龄、车牌号、批次号、溯源码长度/格式等）未校验。

控件类型不匹配：年龄、手机号、车牌等使用了普通 el-input；应使用更合适的控件/限制（如 el-input-number、type="tel"、maxlength、show-word-limit、pattern）。

控件类型不匹配：未限制可选时间范围（如生产/采摘不能晚于当前时间，销售时间不早于存入时间），缺少快捷选项与时区一致性处理。

角色切换时的字段清理：userType 变化时未重置/清空不相关分支的数据，容易残留脏数据。

v-show 使用：角色分支用 v-show 持续渲染隐藏节点，影响性能与校验；更合适用 v-if 按需渲染。

上传图片流程：仅禁用自动上传，缺少文件类型/大小/维度校验、数量限制（limit=1）、错误提示、进度提示与重复选择处理。

预览 URL 内存释放：URL.createObjectURL 后未在文件更换或组件销毁时 URL.revokeObjectURL，有内存泄漏风险。

el-upload 参数不规范：action="#" 容易误导；应统一采用手动上传或自定义请求方式，去掉无效 action。

提交按钮状态：提交时未禁用按钮/表单，可能重复提交；loading 状态与表单交互未联动。

错误处理与提示：后端失败只提示“上链失败”，未展示具体错误信息；catch 未提示用户。

统一的表单样式：大量行内样式（width/label-width）重复，且与 el-form 的 label-width 冲突；应统一到表单或类。

无占位与清空能力：输入缺少 placeholder、clearable/show-password 等基础易用性配置。

slot 用法不当：页面根部的 <span slot="footer"> 不在 el-dialog 中，slot 无效，语义不清且影响结构。

代码可维护性：switch (userType) 构建 FormData 重复性高，可抽象映射；async/await 可读性优于混合 then/catch。

多余/未用字段：Farmer_input.Fa_img 未使用；console.log(formData)/枚举 entries 在生产中应移除或加条件编译。

国际化与可维护性：所有文案硬编码中文，未接入 i18n；不利于后续多语言或统一修改。

响应式布局：表单固定宽度 500px/200px，不适配小屏；缺少 el-row/el-col 栅格与断点控制。

无表单重置：缺少“重置/清空”按钮与对应逻辑，提交成功后也未清空/跳转。

可达性/无障碍：缺少 aria 属性、label 关联与键盘可达性优化。

业务逻辑校验：对“溯源码”在不同角色的必填/只读策略未明确（供应商是否自动生成，其他角色是否强制填写）。

上传必填策略：非“零售商”是否强制上传图片的规则未定义；提交前未校验有无图片。

电话/联系方式：制造商/经销商电话无格式与长度校验；国家码、分机等情况未考虑。

时间字段语义：供应商“原料到货时间”命名与业务语义可能不一致（更可能是“采摘/到厂时间”），需确认。

交互提示：缺少分组标题/分割线（如 el-divider）与字段说明/tooltip，理解成本高。

安全与输入净化：未做前端层面的字符串净化与长度限制，存在 XSS/超长输入风险。

表单提交后体验：成功仅 toast，不提供复制溯源码/跳转详情/查看交易链接等后续动作。

构建与测试：缺少对应的单元测试（表单校验、提交流程、角色切换）与 e2e 冒烟测试。

可用性与交互

输入限制过严：trace 码只允许数字（replace(/[^\d]/g, '')）。若业务允许字母或其他字符，这会导致合法输入被截断。

提交交互不完善：未支持回车触发查询（缺少 @keyup.enter），无清空/重置按钮，无查询历史或最近查询的辅助。

加载过程无反馈：data 里有 loading，但页面未使用 v-loading、按钮禁用态或骨架屏，用户容易误以为无响应。

空数据态缺失：tracedata 为空时没有空态提示或“未找到结果”说明。

时间/数据未格式化：区块链时间/时间字段直接原样显示，无格式化（本地化/相对时间）。

列标题与数据语义疑似不一致：“产品生产时间”列使用的是 farmer_input.fa_pickingTime，语义可能不匹配。

健壮性与错误处理

Promise 缺少 catch：created 的 getFruitList、AllFruitInfo、FruitInfo 的链路都缺少统一的 .catch 或 try/catch 包装，网络异常时用户无反馈。

JSON.parse 风险：多处直接 JSON.parse(res.data) 未做 try/catch；且如果后端已经返回对象，会抛错。应统一服务层返回结构，前端避免重复解析。

深层属性未判空：模板中大量 props.row.farmer_input.fa_xxx 直取，若后端缺字段会报错（Cannot read property of undefined）。建议：
为 tracedata 的每项提供完整默认结构，或使用安全访问（Vue2 可通过计算属性/辅助函数兜底，或在模板里先判断对象存在）。

路由参数变化未响应：仅在 created 读取 this.$route.params.traceability_code，没有 watch 路由变化；从其他页面跳到不同溯源码时不会自动刷新。

状态不一致：FruitInfo 中 tracedata 先置为 [] 再赋值 [0]，期间 UI 闪烁；建议统一以“结果数组”替换并配合 loading。

性能与数据量承载

“获取所有产品信息”高风险：getAllFruitInfo 没有分页/限制，可能一次返回大量数据，表格和展开区渲染开销很大。

展开行渲染重量级：展开区包含大量 el-form-item，数据多时会卡顿；建议虚拟列表/惰性加载展开内容，或分区折叠。

图片加载与预览：多张图片直接加载，无懒加载/占位；建议 lazy、缩略图、小图占位、点击查看大图。

反复构建 URL：字符串模板频繁重复，可抽取方法统一处理并进行 encode（见“安全”）。

安全细节

新窗口打开缺少防护：所有 未加 rel="noopener noreferrer"，存在反向标签劫持风险。

文件名未编码：getImg/${xxx} 未做 encodeURIComponent，特殊字符/注入风险（尽管 Vue 默认转义，URL 仍应编码）。

输入已做长度限制与清洗，较好；但建议对后端返回的所有可插入到 DOM 的字符串进行二次校验或转义显示。

代码质量与可维护性

命名与风格不一致：方法名 FruitInfo、AllFruitInfo 用 PascalCase；JS 通常使用 camelCase（fruitInfo、allFruitInfo）。data 中 input 命名也偏泛化，建议 traceCode。

未使用的计算属性：mapGetters('name','userType') 未在页面使用，应移除以减噪。

重复/冗余样式：.demo-table-expand 定义重复两次；部分样式使用内联 style（如输入框宽度、间距），建议用类统一管理。

魔法数与写死样式：300px 宽度、15px 间距等应抽为样式变量；移动端/响应式体验较差。

表格列缺少排序/筛选能力：如按时间排序等，常见需求未提供。

API 约定不统一：有的接口返回 JSON 字符串需要手动 JSON.parse，有的可能返回对象；应在 api 层统一封装解析与错误处理，视图层只消费标准对象。

使用 FormData 仅传一个简单字段 traceability_code，有点过度；常规 JSON 更轻量，除非后端必须。

i18n 与可访问性（a11y）

i18n 使用不一致：仅 placeholder 做了 $t 兜底判断，大量标签、按钮、标题文本为中文硬编码；应全面接入 i18n。

颜色信息缺乏语义标识：三段标题仅靠颜色区分（绿色/蓝色/橙色/灰色），对色弱不友好；建议用标题层级或图标辅助。

图片无替代文本：el-image 未设置 alt 或 aria-label，影响无障碍。

链接下载体验：图片“点击下载”实际是打开新窗口；可以增加 download 属性或使用更明确的下载按钮。

表单语义：展开区是展示而非可编辑表单，使用 el-form-item 仅为布局；可考虑更语义化/轻量的描述列表组件。

可落地的改进建议（概述）

交互与状态

支持回车查询；使用 loading 控制 v-loading、按钮禁用；增加空态/错误态文案。

添加重置按钮；格式化时间显示（本地化）。

健壮性

在 api 层统一处理 JSON.parse、错误码和异常；视图层只写 try/catch 最外层或依赖统一的 error handler。

为 tracedata 的每项提供默认对象结构，或在渲染前做数据整形。

监听路由参数变化，自动触发查询。

性能

对 getAllFruitInfo 增加分页/限制与服务器端分页；展开区惰性渲染；图片懒加载。

安全

给所有 target="_blank" 的链接加 rel="noopener noreferrer"；图片文件名使用 encodeURIComponent。

代码质量

统一命名风格为 camelCase；移除未使用的 getters；合并重复样式，改内联为类；抽离公共 URL 构建工具。

i18n/a11y

全量替换硬编码文本为 $t('...')；为图片添加 alt；用图标/标签替代单纯颜色区分；可选加键盘可达性支持。
