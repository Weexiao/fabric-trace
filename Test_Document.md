# Fabric Trace 测试文档

## 1. 目的
验证 Fabric Trace 在用户管理、溯源上链、查询、文件处理等核心场景的正确性、可靠性与安全性，覆盖正向与异常路径。

## 2. 范围与对象
- 链下后端 API（Gin + MySQL + Fabric SDK）。
- 链上链码业务逻辑（RegisterUser、Uplink、查询）。
- 前端主要业务流（注册/登录、各角色上链、查询）。
- 文件上传/下载与图片访问。

## 3. 测试策略
- 单元测试：后端工具函数、链码核心逻辑（可用 shim/mock）。
- 集成测试：后端 API ↔ Fabric 网络交互（使用本地测试网络）。
- 系统/E2E：从注册、登录到多角色上链、查询全流程（可用 Postman/接口自动化）。
- 异常与安全：鉴权、非法参数、溯源码校验、文件异常、权限边界。

## 4. 测试环境
- OS：Linux（推荐 Ubuntu）。
- 依赖：Docker & Docker Compose、Go 1.14+、Node.js/NPM、Fabric 本地网络已启动并已部署链码 `trace`。
- 配置：后端 `settings/config.yaml` 正确指向 Fabric 与 MySQL；前端指向后端网关；文件存储目录具备读写权限。

## 5. 角色与前置条件
- 角色：原料供应商、制造商、物流承运商、经销商（或其他扩展角色）。
- 前置：已为各角色创建用户或通过用例 TC01 注册；持有有效 JWT 后再调用受保护接口。

## 6. 测试用例
### 6.1 用户与鉴权
| ID | 场景 | 前置 | 步骤 | 预期 |
|----|------|------|------|------|
| TC01 | 注册成功 | 网络与链码可用 | POST `/register` 提交 `username,password,userType` | 200，返回 `userID`、`txid`，MySQL 与链上均有记录 |
| TC02 | 注册重复用户名 | 同上 | 重复用户名注册 | 提示失败，错误信息包含重复/冲突 |
| TC03 | 登录成功 | TC01 | POST `/login` 正确凭据 | 200，返回 JWT 和 `userType` |
| TC04 | 登录失败-密码错 | TC01 | 错误密码 | 返回失败信息，不生成 JWT |
| TC05 | JWT 鉴权 | TC03 | 携带 JWT 访问受保护接口 | 通过；不带或无效 JWT 返回鉴权失败 |

### 6.2 溯源上链主流程
| ID | 场景 | 前置 | 步骤 | 预期 |
|----|------|------|------|------|
| TC10 | 原料供应商首上链 | 登录角色=原料供应商 | POST `/uplink` 提交原料信息（不传 `traceabilityCode`） | 200，生成 18 位溯源码，链上存 RawSupplierInput，返回 `traceabilityCode` |
| TC11 | 制造商补链 | TC10 溯源码，登录角色=制造商 | POST `/uplink` 携带 TC10 溯源码与制造商字段 | 200，链上产品包含 ManufacturerInput，溯源码一致 |
| TC12 | 物流承运补链 | TC11 完成，登录角色=物流 | 同上，提交物流字段 | 200，CarrierInput 填充 |
| TC13 | 经销商补链 | TC12 完成，登录角色=经销商 | 同上，提交经销商字段 | 200，DealerInput 填充 |

### 6.3 查询与历史
| ID | 场景 | 前置 | 步骤 | 预期 |
|----|------|------|------|------|
| TC20 | 查询产品信息 | 已存在溯源码 | POST `/getIndustrialProductInfo` 携溯源码 | 200，返回完整 IndustrialProduct JSON |
| TC21 | 查询用户产品列表 | 用户已上链 | POST `/getIndustrialProductList` | 200，返回列表含溯源码 |
| TC22 | 查询全部产品 | 需鉴权 | POST `/getAllIndustrialProductInfo` | 200，返回所有产品集合 |
| TC23 | 查询历史 | 已存在产品 | POST `/getIndustrialProductHistory` | 200，返回包含 TxId、Timestamp 的历史数组 |

### 6.4 文件与图片
| ID | 场景 | 前置 | 步骤 | 预期 |
|----|------|------|------|------|
| TC30 | 上链携带图片 | 登录且可写 | POST `/uplink` 携带 `file` 字段 | 200，文件存 `files/images/`，链上记录文件名/哈希 |
| TC31 | 获取图片 | TC30 | GET `/getImg/:filename` | 200，返回图片流 |
| TC32 | 非法文件名 | 无 | GET `/getImg/not_exist` | 404，提示 file not found |

### 6.5 异常与安全
| ID | 场景 | 前置 | 步骤 | 预期 |
|----|------|------|------|------|
| TC40 | 无效溯源码上链 | 非原料角色 | `/uplink` 传不存在或长度≠18 的溯源码 | 返回 "请检查溯源码是否正确!!" |
| TC41 | 未登录访问受保护接口 | 无 JWT | 访问 `/uplink` 等 | 返回鉴权失败 |
| TC42 | 篡改 JWT | 伪造/过期 | 携伪造 JWT | 返回鉴权失败 |
| TC43 | 链码调用失败 | 模拟背书/提交失败 | 触发错误 | 后端返回错误信息，code≠200 |
| TC44 | 重复上链同角色 | 已完成该角色补链 | 再次提交 | 视链码策略：覆盖或拒绝，需验证行为一致性 |

## 7. 自动化建议
- 后端：使用 Go `testing` + httptest，Mock Fabric SDK 或连本地测试网；对 `pkg` 工具函数做单测。
- 链码：使用 Fabric shim mock 进行单元测试，覆盖 `RegisterUser`、`Uplink` 分支、历史查询。
- 前端：Jest/Playwright 做接口与关键路径 E2E（登录、上链、查询）。
- 接口回归：Postman/Newman 集合，分环境变量（dev/test）。

## 8. 测试数据与清理
- 用户名应唯一；密码使用弱口令/强口令各一组测试。
- 溯源码由原料环节生成，后续环节复用；测试结束可重置链码状态或重新部署测试网络。
- 文件上传后可定期清理 `files/uploadfiles` 与 `files/images`，避免磁盘膨胀。

## 9. 风险与关注点
- 当前密码存储方式需确认是否加盐与强哈希；若无则为安全风险。
- 溯源码长度校验固定 18 位，需与前端/生成策略保持一致。
- 文件名直接暴露，建议未来加鉴权或签名 URL。
- Fabric 网络异常（背书、排序服务）会直接影响写入；需在测试中包含网络波动场景。

## 10. 覆盖矩阵（概要）
- 用户：注册/登录/鉴权（TC01-TC05）。
- 上链：四角色流程与分支（TC10-TC13, TC40, TC44）。
- 查询：信息、列表、历史、全量（TC20-TC23）。
- 文件：上传/获取/错误（TC30-TC32）。
- 安全：JWT/无效参数/链码失败（TC40-TC43）。
